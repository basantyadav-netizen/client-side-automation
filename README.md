# New-Employee Onboarding — Claude Code Agents (macOS)

A small agent pipeline for Claude Code that onboards a new Mac:

1. **machine-configurer** → captures RAM, machine name, SSD, OS/version, user → `machine_config.json`
2. **xcode-installer** → installs/verifies Xcode Command Line Tools
3. **homebrew-installer** → installs/verifies Homebrew (+ PATH)
4. **git-configurer** → installs git, sets global `user.name` / `user.email` from `basic_info.json`

A **onboarding-orchestrator** agent runs these in order and stops on the first failure.
A `/onboard` slash command kicks the whole thing off.

## Structure

```
your-project/
├── .claude/
│   ├── settings.json                 # scoped permissions (so it runs without prompts)
│   ├── agents/
│   │   ├── onboarding-orchestrator.md # master: sequences the 4 below, stop-on-fail
│   │   ├── machine-configurer.md
│   │   ├── xcode-installer.md
│   │   ├── homebrew-installer.md
│   │   └── git-configurer.md
│   └── commands/
│       └── onboard.md                # /onboard slash command
├── basic_info.json                   # git identity (edit this!)
└── machine_config.json               # generated by the run
```

## Setup

1. Copy the `.claude/` folder and `basic_info.json` into your project root.
2. Edit `basic_info.json` with the new hire's name and Gmail.
3. **Restart Claude Code** (agent files are loaded at session start; edits on disk need a restart).
4. Confirm the agents loaded: run `/agents` — you should see all five.

## Run it

```
/onboard
```

Or pass the identity inline (the command will write `basic_info.json` for you):

```
/onboard "Jane Doe" jane.doe@gmail.com
```

You can also trigger pieces in natural language, e.g.
`Use the homebrew-installer subagent to install Homebrew`.

## Important notes

- **Nested subagents require Claude Code v2.1.172+.** The orchestrator uses the `Agent`
  tool to spawn the four specialists. Check your version with `claude --version`.
  - **Fallback for older versions:** delete `onboarding-orchestrator.md` and instead let
    the **main session** do the sequencing. Edit `.claude/commands/onboard.md` so the
    Task section reads: *"Run these subagents in order, stopping on the first failure:
    machine-configurer, then xcode-installer, then homebrew-installer, then
    git-configurer."* The main thread will delegate to each leaf agent directly — same
    result, no orchestrator middle-man.
- **Permissions.** `Tasks` install software, so without the included `settings.json`
  you'll get a prompt per command. The provided allow-list scopes exactly the commands
  used. For fully unattended runs you *can* use `--dangerously-skip-permissions` or a
  `permissionMode` of `bypassPermissions`, but prefer the allow-list — it's auditable
  and far safer.
- **Data handoff between agents is via files**, not memory: each subagent runs in an
  isolated context and only returns a short summary. `machine_config.json` and
  `basic_info.json` on disk are the shared state. That's why every specialist ends with
  a `STATUS: PASS|FAIL` block the orchestrator can read.
- **macOS only.** Xcode CLT and Homebrew are macOS-specific; the orchestrator checks
  `uname -s` is `Darwin` and aborts otherwise.
- **Sudo / GUI:** a first-time Xcode CLT install can pop a GUI dialog and Homebrew's
  first install needs `sudo`. For truly hands-off provisioning, pre-authorize sudo and
  read the unattended notes in `xcode-installer.md`.
- **"Xcode" = Command Line Tools** here (the scriptable toolchain git/Homebrew need).
  Full Xcode.app can't be installed from a plain shell — it needs the App Store or `mas`
  with a signed-in Apple ID. See the scope note in `xcode-installer.md`.

## Model choices (tune to taste)

- orchestrator → `opus` (coordination/judgment)
- machine-configurer → `haiku` (cheap, mechanical)
- installers + git → `sonnet` (install/verify logic)

Change the `model:` field in any agent's frontmatter, or set
`CLAUDE_CODE_SUBAGENT_MODEL` to override all of them at once.
